Joinreq


Creates a join request for a group and prints it to standard output.


The join request can be provided to the issuer to join a group as part of dynamic provisioning. joinreq can be found in _install/bin. It can be built using the instructions in Building from Source.


Syntax

joinreq [OPTION] Group Nonce


Positional Arguments

Group : Specifies a file containing the group certificate or group public key.


Nonce : Specifies a file containing the Nonce from the issuer.


Optional Arguments

-f, --privatef : Specifies a file containing the private key f value to use.


-R, --randfile : Specifies a file containing at least 48 random bytes.


-c, --cacert : Specifies the file containing the IoT issuing CA certificate that should be used to verify the inputs.


-h, --help : Displays usage information to standard output.


Remarks

  • If you don't specify --privatef a private f will be generated in an implementation specific manner.
  • If you don't provide --randfile a pseudo random number generator will be used. In production environment you should provide a cryptographically secure random number for generating join requests.
  • If you do not provide --cacert group will be interpreted as an unsigned group public key instead of a group certificate.

Mprecmp


Creates the member pairing precomputation blob and prints it to standard output.


The most computationally expensive operation required during Intel® EPID signature creation is the set of several pairing operations. Fortunately, for any specific member private key, the member precomputation blob can be computed and stored before the first signing request is received. Once the blob is created, it can be used repeatedly across all signing operations against this member private key.

It is essential that the member precomputation blob be stored with integrity protection.

mprecmp can be found in _install/bin. It can be built using the instructions in Building from Source.

For an example of using the Member precomputation blob see the singmsg example in Signing and Verification Tutorial

Syntax

mprecmp [OPTION] Group Member


Positional Arguments

Group : Specifies a file containing the group certificate or the group public key.


Member : Specifies a file containing the member private key or the membership credential.


Optional Arguments

-c, --cacert : Specifies the file containing the IoT issuing CA certificate that should be used to verify the group certificate.


-h, --help : Displays usage information to standard output.


Remarks

  • If you do not provide --cacert Group will be interpreted as a non-signed group public key instead of a group certificate.
